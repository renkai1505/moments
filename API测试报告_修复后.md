## 发现的问题及修复

### 🐛 首页无限跳转Bug
**问题描述**: 访问首页时出现无限跳转的问题

**根本原因**: 
1. 首页(`/pages/index.vue`)在`onMounted`时调用需要认证的API (`/api/child/list`, `/api/growth/list`)
2. 这些API返回错误码4（Token缺失）
3. `useMyFetch`函数在收到错误码3或4时会强制跳转到首页
4. 形成无限循环：首页加载 → API调用失败 → 跳转到首页 → 再次加载 → 无限循环

**修复方案**:
1. **修改`front/utils/index.ts`**: 在跳转逻辑中增加判断，防止在首页时再次跳转
```javascript
if (res.code === 3 || res.code === 4) {
  global.value.userinfo = {}
  // 防止首页无限跳转
  if (process.client && window.location.pathname !== "/") {
    window.location.href = "/"
  }
  throw new Error(res.message || "请求失败")
}
```

2. **修改`front/pages/index.vue`**: 优化错误处理，避免显示认证错误提示
```javascript
catch (error) {
  console.error('加载失败:', error)
  // 不为认证问题显示错误提示
  if (!error.message?.includes('Token缺失')) {
    // 处理其他类型错误
  }
}
```

**修复结果**: ✅ 首页现在可以正常访问，不再出现无限跳转
